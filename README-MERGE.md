# วิธีอัปเดตระบบ VaxKolok เดิม ให้ตอบคำถามเรื่องวัคซีน + แจ้งเตือนจากเว็บไซต์

ไฟล์ในโฟลเดอร์นี้คือ `server.js` เดิมของคุณ **ที่แก้ไขเพิ่มเติมแล้ว** (ของเดิมทุกฟังก์ชัน: ลงทะเบียน HN, ยืนยันตัวตน, ติดตามอาการหลังฉีดวัคซีน, auto follow-up, resend-followup **ยังทำงานเหมือนเดิมทุกอย่าง ไม่มีอะไรหาย**) พร้อมของใหม่ที่เพิ่มเข้าไป 3 อย่าง:

1. **ตอบคำถามเรื่องวัคซีน** — เมื่อผู้ใช้พิมพ์ข้อความที่ไม่ตรงกับคำสั่งที่มีอยู่ (ลงทะเบียน/ยืนยัน/อาการ) ระบบจะค้นคำตอบที่ใกล้เคียงที่สุดจาก `knowledge-base.md` ให้เอง **ไม่ใช้ AI ภายนอก ไม่มีค่าใช้จ่าย ไม่ต้องมี API key ใด ๆ**
2. **`POST /api/notify`** — endpoint ให้เว็บไซต์เรียกมาสั่งส่งข้อความเข้า LINE ได้ (ดูตัวอย่างโค้ดด้านล่าง)
3. **ตั้งเวลาส่งประกาศอัตโนมัติ** (ไม่บังคับ) — ผ่าน `ANNOUNCE_CRON` / `ANNOUNCE_MESSAGE`

และแก้ปัญหาความปลอดภัย: **เอา Channel access token ที่ฝังตรง ๆ ในโค้ดออกแล้ว** ต้องตั้งผ่าน environment variable เท่านั้น

> หมายเหตุ: ระบบตอบคำถามใช้วิธีเทียบความคล้ายของข้อความกับคำถามในฐานความรู้ (ไม่ใช่ AI สร้างคำตอบเอง) ถ้าไม่พบคำถามที่ใกล้เคียงพอ บอทจะแจ้งว่าไม่มีข้อมูลและแนะนำให้ปรึกษาแพทย์แทน จะไม่มีการเดาหรือแต่งคำตอบขึ้นมาเอง

## สิ่งที่ต้องทำ (ทำตามลำดับ)

### 1. ออก Channel access token ใหม่ (สำคัญมาก)

โทเค็นเดิมที่เคยฝังอยู่ในไฟล์ `server.js` ถือว่าหลุดออกมาแล้ว (เพราะไฟล์นี้ถูกอัปโหลด/แชร์ออกมา) ให้ไป:

1. LINE Developers Console → Channel ของคุณ (VaxKolok-DHB) → แท็บ **Messaging API**
2. ที่หัวข้อ Channel access token กด **Issue** ใหม่อีกครั้ง (โทเค็นเก่าจะใช้ไม่ได้ทันที ปลอดภัยขึ้น)
3. คัดลอกค่าใหม่ไว้ใช้ในขั้นตอนที่ 3

### 2. เตรียมไฟล์

คัดลอกไฟล์ต่อไปนี้จากโฟลเดอร์นี้ไปวางในโปรเจกต์เดิมของคุณ (ตำแหน่งเดียวกับ `server.js` เดิม):

```
your-project/
├── server.js              ← แทนที่ไฟล์เดิมด้วยไฟล์ในโฟลเดอร์นี้
├── knowledge-base.md       ← ไฟล์ใหม่ วางไว้ระดับเดียวกับ server.js
└── ai/
    ├── answer.js            ← ไฟล์ใหม่
    ├── kbSearch.js           ← ไฟล์ใหม่ (ระบบค้นคำตอบ)
    └── knowledgeBase.js      ← ไฟล์ใหม่
```

จากนั้นติดตั้งไลบรารีที่เพิ่มเข้ามาใหม่ (รันในโปรเจกต์เดิมของคุณ ไม่ต้องลบ dependency เดิม):

```bash
npm install node-cron
```

> ถ้าคุณยังไม่มี `package.json` ในโปรเจกต์เดิมเลย ใช้ไฟล์ `package.json` ในโฟลเดอร์นี้แทนได้เลย (มี dependency ครบทั้งของเดิมและของใหม่)

### 3. เพิ่ม Environment Variables บน Render

ไปที่ Render → service ของคุณ (vaccine-line-api) → **Environment** แล้วเพิ่ม/แก้ไขค่าต่อไปนี้:

| ตัวแปร | ค่า | หมายเหตุ |
|---|---|---|
| `LINE_TOKEN` | โทเค็นใหม่จากขั้นตอนที่ 1 | **ต้องตั้งค่านี้ ไม่งั้นบอทส่งข้อความไม่ได้** |
| `DB_URL` | (มีอยู่แล้ว ไม่ต้องแก้) | Firebase Realtime DB เดิม |
| `NOTIFY_API_KEY` | ตั้งรหัสลับเองยาว ๆ สุ่ม | ให้เว็บไซต์ใช้ยืนยันตัวตนตอนเรียก `/api/notify` |
| `ANNOUNCE_CRON` | เช่น `0 9 * * 1` (ไม่บังคับ) | ถ้าต้องการประกาศอัตโนมัติทุกสัปดาห์ |
| `ANNOUNCE_MESSAGE` | ข้อความประกาศ (ไม่บังคับ) | คู่กับ `ANNOUNCE_CRON` |

ไม่ต้องตั้งค่าเกี่ยวกับ AI/API key ใด ๆ อีกแล้ว บันทึกแล้ว Render จะ deploy เวอร์ชันใหม่ให้อัตโนมัติ

### 4. ทดสอบ

- พิมพ์คุยกับบอทด้วยคำถามทั่วไป เช่น "ลูกฉีดวัคซีนแล้วมีไข้ทำอย่างไร" → ควรได้คำตอบจากฐานความรู้
- ถ้าถามเรื่องที่ไม่มีในฐานความรู้ บอทจะแจ้งว่าไม่มีข้อมูลและแนะนำให้ปรึกษาแพทย์ (ไม่เดาคำตอบ)
- พิมพ์ `ลงทะเบียน 12345` → ควรทำงานเหมือนเดิมทุกอย่าง (ระบบเช็คคำสั่งนี้ก่อนเสมอ ไม่ถูกแย่งตอบ)
- ทดสอบ `/api/notify` จากเครื่องตัวเอง:
  ```bash
  curl -X POST https://vaccine-line-api.onrender.com/api/notify \
    -H "Content-Type: application/json" \
    -H "x-api-key: ค่า NOTIFY_API_KEY ของคุณ" \
    -d '{"message":"ทดสอบระบบแจ้งเตือน","mode":"broadcast"}'
  ```

## โหมดการส่งของ `/api/notify`

| mode | ความหมาย |
|---|---|
| `broadcast` (ค่าเริ่มต้น) | ส่งหาเพื่อนของ LINE OA ทุกคน |
| `registered` | ส่งเฉพาะผู้ปกครองที่เคยลงทะเบียน HN สำเร็จแล้ว (ดึง `lineUserId` จาก Firebase `children`) |
| `push` (ต้องแนบ `userId`) | ส่งหาผู้ใช้รายเดียว |

ตัวอย่างเรียกจากเว็บไซต์ (JavaScript):

```js
fetch('https://vaccine-line-api.onrender.com/api/notify', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json', 'x-api-key': 'ค่า NOTIFY_API_KEY ของคุณ' },
  body: JSON.stringify({
    message: 'เปิดจองคิวฉีดวัคซีนไข้หวัดใหญ่รอบใหม่แล้ว',
    mode: 'registered', // ส่งเฉพาะผู้ปกครองที่ลงทะเบียนไว้กับระบบ
  }),
});
```

⚠️ อย่าเรียก `/api/notify` จาก JavaScript ฝั่งเบราว์เซอร์ตรง ๆ (จะเห็น `NOTIFY_API_KEY` ได้) ให้เรียกจากฝั่ง backend ของเว็บไซต์แทน
